# Reference Image Chaining + Hex Palette

**Implemented:** 2026-02-15
**Status:** Tested and verified (3/3 designs passed)

## Problem

Each AOP hoodie area (front, back, sleeve) was generated by independent Gemini calls with no visual reference between them. The only coherence mechanism was Kimi K2.5 writing prompts with the same palette/style words, but Gemini interprets "gold on navy" differently every time, producing mismatched colors across areas.

## Solution

Two complementary mechanisms for color/style coherence:

### 1. Reference Image Chaining (Primary)

After generating the front panel, pass it as a reference image to all subsequent Gemini calls (back, sleeve). Gemini receives both the text prompt and the front image, so it can match the exact colors and rendering style.

```
Before (no visual coherence):
  Front  -> Gemini(text_only)          -> gold #1, navy #1
  Back   -> Gemini(text_only)          -> gold #2, navy #2
  Sleeve -> Gemini(text_only)          -> gold #3, navy #3

After (visual anchor):
  Front  -> Gemini(text_only)          -> establishes exact palette
  Back   -> Gemini(text + front_image) -> matches front's exact colors
  Sleeve -> Gemini(text + front_image) -> matches front's exact colors
```

The front image is downscaled to ~1024px before passing as reference (Gemini doesn't need 4K, saves bandwidth).

### 2. Hex Palette in Design Plan (Belt-and-suspenders)

Kimi K2.5 now outputs exact hex color codes in every area prompt (e.g., "deep navy (#1B2438) with gold accents (#C9A94E)") and a top-level `hex_palette` dict in the plan JSON. This gives Gemini explicit color targets in addition to the visual reference.

## Files Modified

### `prototype/asset_pipeline/generator.py`
- Added `reference_image: Image.Image | None = None` parameter to `generate_design()`
- When provided, `contents` becomes `[prompt, reference_image]` instead of just the prompt string
- Gemini SDK accepts PIL Images directly in contents list

### `prototype/design_director/models.py`
- Added `hex_palette: dict[str, str] = field(default_factory=dict)` to `DesignPlan`
- Stores named hex colors: `{"primary": "#1B2438", "secondary": "#C9A94E", "accent": "...", "background": "..."}`

### `prototype/design_director/prompts.py`
- Added rule 9 to `PLAN_SYSTEM_PROMPT`: requires hex color codes in every area prompt and `hex_palette` field in output
- Added `hex_palette` field to the JSON output schema
- Added `REFERENCE_IMAGE_INSTRUCTION` constant: "Match the exact colors, rendering style, and atmospheric mood of the provided reference image."

### `prototype/design_director/director.py`
- Parses `hex_palette` from Kimi's JSON response into `DesignPlan.hex_palette`
- New function `inject_reference_instruction(prompt: str) -> str` that prepends the reference matching instruction
- Imports `REFERENCE_IMAGE_INSTRUCTION` from prompts.py
- Prints hex palette in plan summary output

### `prototype/8_design_director_pipeline.py` (CLI)
- New helper `_downscale_for_reference(img, max_px=1024)` for bandwidth-efficient reference passing
- `generate_all_images()` implements dependency chain:
  - First area: text-only generation (the "anchor") — captures raw image as `anchor_image`
  - Subsequent areas: prepends reference instruction to prompt, passes downscaled anchor as `reference_image`
- `display_plan()` shows hex palette
- `plan.json` serialization includes `hex_palette`

### `prototype/9_director_ui.py` (Streamlit)
- Same reference image chaining as CLI in Step 3 (Generate Images)
- Captures `anchor_image` after first generation
- Downscales reference inline (~1024px)
- Plan review step displays hex palette
- `plan.json` serialization includes `hex_palette`

## Backward Compatibility

- `reference_image` defaults to `None` — existing code calling `generate_design()` without it works unchanged
- `hex_palette` defaults to `{}` — plans generated by older Kimi prompts parse fine (empty dict)
- No changes to Printify upload, mockup download, or review loop

### Performance Notes

- Reference image adds negligible overhead (~0s — Gemini processes the 824x1024 reference inline)
- Back/sleeve generation times are comparable to front (no slowdown from multimodal input)
- Total pipeline time per design: ~4-5 min (plan + 3 images + 7 uploads + product creation + mockups)

## Key Technical Details

- Gemini SDK accepts PIL Images directly in `contents` list — no base64 conversion needed
- Reference image is captured from raw generation output (before bg removal/resize) for best color fidelity
- `inject_reference_instruction()` lives in `director.py` (not prompts.py) because it's a runtime transform, not a template
- The anchor is always the first area in `generation_order` (currently "front" for AOP hoodie)
