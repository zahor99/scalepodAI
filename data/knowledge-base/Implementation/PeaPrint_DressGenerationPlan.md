     │                                                                                                                                                                                                           │
     │ Phase 0: Infrastructure (do once)                                                                                                                                                                         │
     │                                                                                                                                                                                                           │
     │ 0.1 Add PeaPrint as supplier type in the pipeline                                                                                                                                                         │
     │                                                                                                                                                                                                           │
     │ Files to modify:                                                                                                                                                                                          │
     │ - prototype/suppliers/database.py — Add PeaPrint blank entries with printify_blueprint_id: 0, printify_provider_id: 0 and brand: "PeaPrint"                                                               │
     │ - prototype/pricing/suppliers.py — Add PeaPrint product entries with COGS (including 34.5% tariff) and niche pricing                                                                                      │
     │ - prototype/design_director/niche_loader.py — No change needed (niche detection is keyword-based)                                                                                                         │
     │                                                                                                                                                                                                           │
     │ Key difference from Yoycol: PeaPrint uses a single uniform canvas for all panels (e.g., 3634x4500 for all panels of a product). The garment shape is mask-clipped during production. Our YAML rules       │
     │ should set print_dimensions to the canvas size and note that artwork must fill the entire canvas.                                                                                                         │
     │                                                                                                                                                                                                           │
     │ 0.2 Create a PeaPrint YAML rules template                                                                                                                                                                 │
     │                                                                                                                                                                                                           │
     │ Create prototype/rules/_template_peaprint_dress.yaml as a base template. Each new product copies this and fills in:                                                                                       │
     │ - product_type identifier                                                                                                                                                                                 │
     │ - print_dimensions (single canvas WxH from Airtable)                                                                                                                                                      │
     │ - generation_order (front + back, optionally right_sleeve)                                                                                                                                                │
     │ - trim.areas (binding, collar, cuffs — solid color fill, zero gen cost)                                                                                                                                   │
     │ - mirror directives (left_sleeve from right_sleeve, etc.)                                                                                                                                                 │
     │                                                                                                                                                                                                           │
     │ 0.3 Airtable-to-YAML extraction script                                                                                                                                                                    │
     │                                                                                                                                                                                                           │
     │ Create prototype/tools/airtable_to_yaml.py that:                                                                                                                                                          │
     │ 1. Reads PeaPrint records from Airtable Supplier Catalog via MCP or export                                                                                                                                │
     │ 2. Parses Asset Requirements text to extract panel names, canvas dims, panel count                                                                                                                        │
     │ 3. Auto-classifies panels: generated (front/back/sleeve) vs trim (binding/collar/cuff/hem) vs mirror (left=right)                                                                                         │
     │ 4. Generates YAML rules file from template                                                                                                                                                                │
     │ 5. Generates database.py entry snippet                                                                                                                                                                    │
     │ 6. Generates suppliers.py pricing entry snippet                                                                                                                                                           │
     │                                                                                                                                                                                                           │
     │ This script replaces manual creation for each product.                                                                                                                                                    │
     │                                                                                                                                                                                                           │
     │ 0.4 PeaPrint white base mockup generation                                                                                                                                                                 │
     │                                                                                                                                                                                                           │
     │ For each PeaPrint product type, generate white base mockups for VTON/compositing:                                                                                                                         │
     │ - Use Vertex AI or kie.ai to generate plain white garments in various poses                                                                                                                               │
     │ - Store in prototype/image_producer/bases/base_pp_<type>_*.png                                                                                                                                            │
     │ - Add to COMPOSITING_SLOTS_BY_TYPE in prototype/image_producer/aop_compositing.py                                                                                                                         │
     │ - Add to VTON_TEMPLATES_BY_TYPE in prototype/asset_pipeline/vton_generator.py                                                                                                                             │
     │                                                                                                                                                                                                           │
     │ Cost optimization: Start with 2 bases per dress (ghost front, ghost back) instead of 7-8. Add lifestyle/detail bases only for products that sell well.                                                    │
