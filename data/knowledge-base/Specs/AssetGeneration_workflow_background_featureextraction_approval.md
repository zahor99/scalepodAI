The Root Cause
Each area (front, back, sleeve) is generated by a standalone text→image Gemini call with no visual reference to the other areas. The only coherence mechanism is the Design Director writing prompts with the same palette/style words — but Gemini interprets "gold on navy" differently every time. The front might get warm amber-gold on deep midnight blue, while the sleeve gets pale yellow-gold on bright navy. Same words, different visual interpretation.

The good news: the Gemini SDK already supports reference_images — your pipeline just doesn't use it yet.

Proposal 1: Reference Image Chaining (Highest Impact, Medium Effort)
What: After generating the front panel, pass it as a reference_image to the back and sleeve generation calls. This gives Gemini a visual anchor for colors, style, and mood.

How it changes generator.py:

python
# Current: generate_design(prompt, aspect_ratio, image_size)
# Proposed: generate_design(prompt, aspect_ratio, image_size, reference_image=None)
```

When `reference_image` is provided, the Gemini call becomes multimodal — the model sees the front panel image alongside the text prompt for the sleeve, so it naturally matches the exact color rendering, brush style, and atmospheric feel.

**Generation order becomes a dependency chain:**
1. Front → generated from text only (the "anchor")  
2. Back → generated with front as reference  
3. Sleeve → generated with front as reference  

**Changes needed:**
- `generator.py`: Add `reference_image` parameter, pass it in the Gemini `contents` as `[reference_image, prompt_text]`
- `8_design_director_pipeline.py`: After generating front, store it and pass it to back/sleeve calls
- `prompts.py`: Add instruction to the sleeve/back prompts like "Match the exact colors and rendering style of the provided reference image"

**Risk:** Gemini might over-copy the reference and make the sleeve look too similar to the front (same scene instead of same style). Mitigation: the text prompt still controls *what* is drawn, the reference just anchors *how* it looks.

## Proposal 2: Explicit Hex Color Injection in Prompts (Quick Win, Low Effort)

**What:** After the Design Director picks a palette, force the actual Gemini prompts to include exact hex codes rather than vague color names. Right now the coherence section says "use the EXACT same color palette" but the actual prompts might just say "gold on navy." 

**How it changes `prompts.py`:**

Add a new rule to `PLAN_SYSTEM_PROMPT`:
```
9. CRITICAL COLOR SPECIFICATION: In every area_prompt, you MUST specify the exact 
   hex color codes inline. Do NOT use vague color names alone. Example:
   BAD: "gold accents on a navy background"
   GOOD: "rich gold (#C9A94E) metallic accents on deep navy (#1B2438) background, 
          with cream (#F5F0E1) highlights and deep purple (#2D1B4E) shadows"
   
   The hex codes MUST be identical across all area prompts. Copy-paste them.
Also update the output schema to include a hex_palette field that gets reused:

json
{
  "hex_palette": {"primary": "#1B2438", "secondary": "#C9A94E", "accent": "#F5F0E1"},
  "area_prompts": { ... }
}
Why it helps: Image generation models do respond to hex codes in prompts. It's not perfect, but it narrows the interpretation gap significantly compared to "gold" which could be anything from pale wheat to deep bronze.

Proposal 3: Palette Extraction + Regeneration Loop (Medium Impact, Medium Effort)
What: After generating the front panel, use a vision model (Gemini or Claude) to extract the actual rendered colors (dominant 5-6 hex values), then inject those extracted colors into subsequent area prompts.

Flow:

Generate front panel
Send front panel image to vision model: "Extract the 5 dominant hex color values from this image"
Get back: ["#1A2340", "#D4A84F", "#2C1A50", "#F0EBD8", "#8B7EC0"]
Replace the palette references in the back/sleeve prompts with these actual rendered colors
Generate back and sleeve with the corrected prompts
Changes needed:

New utility function: extract_palette(image) -> list[str]
8_design_director_pipeline.py: Insert palette extraction step after front generation, string-replace palette terms in remaining prompts
director.py: Add a refine_prompts_with_palette() method that takes extracted hex values and rewrites remaining area prompts
Why it's different from Proposal 2: Proposal 2 tells the Design Director to request specific colors. Proposal 3 checks what actually got rendered and feeds that reality back into subsequent prompts. It closes the loop.

Proposal 4: Shared Background Layer + Compositing (High Impact, High Effort)
What: Instead of generating each area as a completely independent scene, generate a single large "background atmosphere" image first (the starfield, the color gradient, the mood), then use it as the literal background for every panel with panel-specific foreground elements composited on top.

Flow:

Generate a seamless background tile (e.g., 4K starfield with gold nebula and navy sky)
For the front: composite the hero subject onto this background
For the back: composite the complementary scene onto the same background
For the sleeve: crop/tile the background, add flowing accent elements
Why: The background is what creates the color and mood consistency. If every panel shares the same literal background pixels, they'll inherently match. The foreground subjects differentiate the panels while the base layer unifies them.

Downside: More complex pipeline, requires image compositing (Pillow/PIL), and the "shared background" approach works best for atmospheric/pattern-based designs (which celestial/boho is — so it might actually work great for this niche). It's less suitable for designs with unique scene-based backgrounds per panel.

Proposal 5: Strengthen the YAML Coherence Rules with Cross-Area Prompt Templates (Quick Win)
What: Add a new coherence.prompt_suffix field to the YAML that gets appended to every area's prompt verbatim. This ensures the exact same color/style language appears in every Gemini call.

Add to aop_hoodie.yaml:

yaml
coherence:
  prompt_suffix: >
    Color palette MUST be exactly: deep navy (#1B2438) background, 
    rich gold (#C9A94E) as primary accent, deep purple (#2D1B4E) 
    as secondary, cream (#F5F0E1) highlights. Art style MUST be 
    consistent: [STYLE_PLACEHOLDER]. All areas of this garment 
    must look like they belong to the same artwork.
Then in build_plan_user_prompt(), inject this suffix into the instructions so the Design Director appends it to every area prompt it generates.
